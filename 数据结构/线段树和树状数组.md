# 线段树与树状数组的区别与联系

## 一、基本概念

- **线段树（Segment Tree）**  
  是一种平衡二叉树结构，主要用于处理**区间查询和区间修改**的问题，适用于数据静态或少量变动的场景。

- **树状数组（Binary Indexed Tree / Fenwick Tree）**  
  是一种基于位运算的高效数据结构，主要用于**前缀和查询与单点修改**，实现相对简单，占用空间较小。

---

## 二、功能对比

| 功能类型     | 线段树                   | 树状数组                |
|--------------|--------------------------|-------------------------|
| 单点更新     | ✅ O(log n)              | ✅ O(log n)             |
| 区间查询     | ✅ O(log n)              | ✅ O(log n)（仅限前缀和）|
| 区间修改     | ✅ O(log n)              | ⚠️ 需特殊技巧支持       |
| 支持的数据类型 | 任意可合并的操作（如最小值、最大值、gcd 等） | 通常是加法，其他操作不易实现 |
| 编码复杂度   | 中等，较长              | 简单，代码短小         |
| 空间复杂度   | O(n) 或 O(4n)            | O(n)                   |

---

## 三、适用场景

- 使用树状数组的场景：
  - 频繁进行**单点更新 + 前缀查询**
  - 对空间或实现复杂度敏感的题目
  - 数据类型是加法或简单前缀性质（如数出现次数）

- 使用线段树的场景：
  - 需要**支持多种区间操作**（如区间最值、gcd、区间最大连续子段和等）
  - 需要**支持区间修改**
  - 需要拓展为**可持久化线段树、动态开点线段树、线段树合并等高级功能**

---

## 四、联系

- 两者的核心思想都是**将整体问题分解为局部子问题**，并通过**树结构合并结果**。
- 树状数组可以看作是一种对线段树的空间压缩和结构简化。
- 在处理某些**特殊限制的区间问题**时，它们可以互相替代。

---

## 五、代码结构比较

### 树状数组（BIT）示例

```cpp
int lowbit(int x) {
    return x & -x;
}

void add(int i, int x) {
    while (i <= n) {
        bit[i] += x;
        i += lowbit(i);
    }
}

int prefixSum(int i) {
    int res = 0;
    while (i > 0) {
        res += bit[i];
        i -= lowbit(i);
    }
    return res;
}
